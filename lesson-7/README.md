## Порівняння стратегій

### RollingUpdate (default)
- **Переваги**: zero/мінімальний downtime; плавний rollout; легкий rollback (`kubectl rollout undo`).
- **Недоліки**: дві версії додатку працюють одночасно — потрібна сумісність схеми БД та API; повільніше.

### Recreate (Replace для Deployment)
- **Переваги**: проста семантика; у кластері одночасно лише одна версія (немає проблем сумісності).
- **Недоліки**: повний downtime між фазами, треба застосовувати інші патерни для zero downtime.


## Зміна ConfigMap
При зміні ConfigMap, що використовується в Deployment, Kubernetes не перезапускає поди автоматично. Щоб застосувати нові налаштування, потрібно вручну перезапустити поди.
